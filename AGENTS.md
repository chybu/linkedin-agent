# Codex Instructions

- Do not modify any code unless the user explicitly asks for code changes.
- If the user asks for a solution, explain it first and provide copyable code snippets only.
- Do not edit files automatically.
- Do not assume permission to patch files.
- If a change is suggested, show exactly what to add, remove, or replace.
- When suggesting git commits, prefer conventional commit prefixes such as `chore:`, `fix:`, `feat:`, or `docs:`.
- For routing, security, or refactoring-only changes, `chore:` is usually appropriate.
- If the user asks for a commit message, give a ready-to-run `git commit -m "..."` example.
